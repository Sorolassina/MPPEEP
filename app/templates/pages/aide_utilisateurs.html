👥 Système de Gestion des Utilisateurs

Le système de gestion des utilisateurs permet de créer et gérer les comptes d'accès à l'application, de définir leurs rôles et permissions, et de les lier aux agents du personnel.

🎭 Types d'Utilisateurs

👑 Administrateur (admin)

  • Accès complet à tous les modules
  • Gestion des utilisateurs et permissions
  • Configuration du système (workflows, référentiels, paramètres)
  • Accès aux logs et statistiques globales

🔧 Modérateur (moderator)

  • Gestion des modules opérationnels (RH, Budget, Stock)
  • Validation des demandes selon les rôles personnalisés assignés
  • Consultation des rapports et statistiques
  • Pas d'accès à la configuration système

👤 Utilisateur (user)

  • Accès limité aux fonctionnalités de base
  • Création de demandes (congés, permissions, besoins)
  • Suivi de ses propres demandes
  • Consultation des informations

➕ Créer un Utilisateur

📝 Méthode 1 : À partir d'un agent existant (recommandé)

  1. Aller dans "Gestion du Personnel" (/api/v1/personnel/)
  2. Identifier l'agent qui n'a pas encore de compte (icône 👤➕ bleue)
  3. Cliquer sur l'icône 👤➕ pour ouvrir le modal de création
  4. Saisir l'email de connexion (pré-rempli avec suggestion)
  5. Définir un mot de passe :
    • Soit saisir un mot de passe personnalisé (min. 8 caractères)
    • Soit cliquer sur "🔄 Générer" pour un mot de passe aléatoire sécurisé
    • Soit laisser vide pour génération automatique côté serveur
  6. Confirmer la création
  7. Noter le mot de passe affiché et le communiquer à l'utilisateur de manière sécurisée

💡 Avantage : Le compte est automatiquement lié à l'agent, permettant de créer des demandes RH et d'utiliser les workflows personnalisés.

🖊️ Méthode 2 : Depuis la gestion des utilisateurs

  1. Aller dans "Gestion des Utilisateurs"
  2. Cliquer sur "➕ Nouvel Utilisateur"
  3. Remplir le formulaire :
    • Email (obligatoire)
    • Nom complet
    • Mot de passe
    • Type d'utilisateur (admin, moderator, user)
    • Lier à un agent (optionnel mais recommandé)
  4. Enregistrer

🔗 Lier un Utilisateur à un Agent

Le lien entre un utilisateur et un agent est essentiel pour :

  • ✅ Permettre à l'utilisateur de créer des demandes RH
  • ✅ Identifier automatiquement le demandeur dans les workflows
  • ✅ Assigner des rôles personnalisés dans les circuits de validation
  • ✅ Afficher les informations personnelles (matricule, service, grade, etc.)

⚠️ Important : Un utilisateur non lié à un agent ne pourra pas créer de demandes RH ni participer aux workflows de validation.

🔐 Gérer les Permissions

Modifier le type d'utilisateur

  1. Dans la liste des utilisateurs, cliquer sur ✏️ Modifier
  2. Changer le champ "Type d'utilisateur"
  3. Enregistrer les modifications

Activer/Désactiver un compte

  • Désactiver : L'utilisateur ne pourra plus se connecter mais ses données sont conservées
  • Réactiver : L'utilisateur peut à nouveau se connecter
  • Supprimer : Suppression définitive du compte (à éviter)

✨ Bonnes Pratiques

  • ✅ Toujours lier les utilisateurs aux agents pour une gestion RH complète
  • ✅ Utiliser des emails professionnels (prenom.nom@ministere.sn)
  • ✅ Générer des mots de passe forts et les communiquer de manière sécurisée
  • ✅ Définir le bon type d'utilisateur selon les responsabilités
  • ✅ Désactiver plutôt que supprimer pour conserver l'historique
  • ✅ Vérifier régulièrement les comptes inactifs

🔧 Dépannage

❌ "Votre compte n'est pas lié à un agent"

Solution : Demander à un administrateur de modifier votre compte utilisateur et de sélectionner votre agent dans le champ "Agent associé".

❌ "Email déjà utilisé"

Solution : Vérifier si un compte existe déjà avec cet email. Si l'utilisateur a oublié son mot de passe, utiliser la fonction de réinitialisation.

❌ "Cet agent a déjà un compte utilisateur"

Solution : L'agent est déjà lié à un compte. Vérifier dans la liste des utilisateurs pour retrouver le compte existant (icône 👤✓ verte).
